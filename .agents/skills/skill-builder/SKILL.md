[任务与边界]
    做：
    - 按 [9 要素维度] + [反模式规则] 创建新 Skill 骨架
    - 输出 9 要素自检报告

    不做：
    - 不修改已有 Skill
    - 不写业务内容（只搭骨架）
    - 不做框架级注册（属跨 Skill 协调，由主 Agent 处理）

    完成标准：
    - 新 SKILL.md 含 [9 要素维度] 必备项
    - [反模式规则] 扫描通过
    - 自检报告已输出

[第一性原则]（优先级从高到低）

    1. 9 要素必备（最高）
       新 SKILL.md 必须含 [9 要素维度] 全部
       缺必备项 → 不允许创建

    2. 反模式不犯
       禁止 [反模式规则] 列出的所有反模式
       详见 [创建知识] [反模式规则]

    3. 模板优先
       从 templates/skill-template.md 开始
       不从零写

    4. 参照现有（学风格不学 bug）
       按 [Skill 类型 - 按通用特征] 找 1-2 个参照 Skill
       用 [反模式规则] 过滤参照里的 bug

[依赖检测]
    必需：
    [1] 创建意图（来源不限）
        - 用户主动需求："我想造个 X Skill"
        - 任何上游机制提议（自进化引擎 / 其他 Agent / 其他 Skill）
        失败话术："skill-builder 需要明确的创建意图，未收到。"
    [2] templates/skill-template.md 模板
        失败话术："templates/skill-template.md 不存在，框架级 bug。"

    可选：
    - 上游提议附带的背景资料（如相关 feedback 记录，由调用方以路径或内容形式提供）

    [路径变量]
    $SKILLS_DIR  = 项目里已存在的 Skill 集合位置（Codex 默认 .agents/skills/）
    $FEEDBACK_DIR = 项目的 feedback 集合位置（Codex 默认 .codex/feedback/）
    其他框架/项目按对应约定提供。

[创建知识]

    内部按"维度 / 规则 / 策略 / 思想"四组组织。

    ━━━ 【维度组 - 看什么】━━━

    [9 要素维度]
    被造的 Skill 必须包含的 9 个 Section（必备 8 + 按需 1）：

    必备 8 项：
    0. frontmatter（name + version + description + depends_on）
    1. [任务与边界]
    2. [第一性原则]
    3. [依赖检测]
    4. [核心知识层]（维度/规则/策略 - 至少一节）
    5. [工作流程]
    6a. [输出格式]
    7. [初始化]

    按需 1 项：
    6b. [输出风格]（业务 Skill 必须，元 Skill 可选）

    [Skill 类型 - 按通用特征]
    判断本次要造的 Skill 属于哪类：

    | 类型 | 通用特征（识别条件）|
    |---|---|
    | 执行型 - 审查 | 输入数据 + 输出报告（不修改输入）|
    | 执行型 - 修复 | 输入问题 + 输出修复后的状态 |
    | 执行型 - 编码 | 输入需求 + 输出代码/文件 |
    | 对话型 - 采访 | 输入用户模糊描述 + 多轮对话 + 输出明确文档 |
    | 外部工具调用型 | 通过外部接口（如 MCP）调用工具完成任务 |
    | 分析型 - 规划 | 读已有文档 + 分析依赖 + 输出规划文档 |
    | 元 Skill | 服务框架自己（不直接服务用户产品）|
    | 元-元 Skill | 创建其他 Skill 的工具 |

    ━━━ 【规则组 - 守什么】━━━

    [反模式规则]（创建时禁止的 12 条）
    ❌ [文件结构] 单文件凑数节
    ❌ [技能] 与 [工作流程] 重复
    ❌ [启动检查] 与 [初始化] 命名冲突
    ❌ [提议格式] + [返回格式] 拆成两节（应合并）
    ❌ [信息充足度判断] 独立节（应嵌入维度）
    ❌ 失败话术含 "请先 /xxx"
    ❌ 流程末尾"接下来 /xxx"
    ❌ [Skill 边界外] / [调用上下文]
    ❌ "联网优先" 做第一性原则
    ❌ depends_on 与 [依赖检测] 不同步
    ❌ 模板字段 vs SKILL [维度] 不对称
    ❌ 模板重复 SKILL 已定义的规则

    [格式规范]
    - Section 标题：[标题]
    - 缩进：4 空格
    - 使用项目主语言（跟随当前项目，中/英/其他）

    [命名规范]
    - Skill 名：kebab-case（如 dev-planner）
    - 目录：$SKILLS_DIR/[skill-name]/
    - 主文件：SKILL.md
    - 模板（如有）：templates/ 子目录

    ━━━ 【策略组 - 怎么做】━━━

    [找参照的策略]
    1. 扫描 $SKILLS_DIR（项目里已存在的 Skill 集合）
    2. 按 [Skill 类型 - 按通用特征] 匹配每个已存在的 Skill
    3. 找到 1-2 个最接近的实际 Skill 作为参照
    4. 如果项目里没有同类型 Skill（如全新项目）→ 跳过参照，直接按 [9 要素维度] 创建

    [按需扩展策略]（按 Skill 类型加节）
    - 对话型 / 分析型 → 维度内嵌"充足标准"
    - 多模式 → [多模式工作流程]
    - 元 Skill → [确认后执行]
    - AI 容易跳过步骤的 Skill → [反合理化清单]

    [条件性联网策略]
    仅以下情况执行 web.run：
    - 涉及不熟悉的领域 → 了解最佳实践
    - 不确定某 section 命名是否符合社区惯例
    其他情况：按 [9 要素维度] + [反模式规则] 创建即可

    ━━━ 【思想组 - 信什么】━━━

    [三层模块化]
    - 第一层：原子能力（Section）
    - 第二层：Skill（SKILL.md）
    - 第三层：工作流（AGENTS.md）

[工作流程]

    第 1 步：需求收集
        了解：
        - 这个 Skill 解决什么问题？
        - 何时触发？
        - 输入输出是什么？
        - 来自上游提议 → 由调用方提供背景资料（如相关 feedback；如需读取 $FEEDBACK_DIR 由调用方说明）

    第 2 步：识别 Skill 类型 + 找参照
        按 [创建知识] [Skill 类型 - 按通用特征] 判定本次要造的 Skill 类型
        按 [创建知识] [找参照的策略] 扫描 $SKILLS_DIR 里的实际 Skill
        - 找到同类型 → 读 1-2 个参照 SKILL.md，参照风格细节，用 [反模式规则] 过滤 bug
        - 找不到同类型（项目刚起步 / 全新领域）→ 跳过参照，直接按 [9 要素维度] 创建

    第 3 步：按 9 要素决定结构
        必备 8 项全列
        按 [按需扩展策略] 加按需扩展节
        决定知识层叫什么（[XXX 维度] / [XXX 规则] / [XXX 策略]）

    第 4 步：填充内容
        从 templates/skill-template.md 开始
        逐节填写
        如涉及陌生领域 → 触发 [条件性联网策略]

    第 5 步：9 要素自检 + 反模式扫描
        用 [创建知识] [9 要素维度] 逐项检查
        用 [创建知识] [反模式规则] 逐条扫描
        有问题 → 回第 4 步修
        全通过 → 进入第 6 步

    第 6 步：写入文件 + 返回报告
        创建 $SKILLS_DIR/[skill-name]/SKILL.md
        如有模板 → 创建 templates/ 子目录
        按 [输出格式] 返回创建报告
        （注：框架级注册由主 Agent 处理，不在本 Skill 范围）

[输出格式]

    创建完成后返回主 Agent：

    """
    🛠 新 Skill 已创建

    Name: [name]
    路径：$SKILLS_DIR/[name]/SKILL.md
    类型：[执行型/对话型/外部工具/分析型/元 Skill]
    参照：[扫描出的实际参照 Skill 名 / 或 "无（全新项目跳过参照）"]

    9 要素自检：
    ✅/❌ 0. frontmatter
    ✅/❌ 1. [任务与边界]
    ✅/❌ 2. [第一性原则]
    ✅/❌ 3. [依赖检测]
    ✅/❌ 4. [创建知识/维度/策略]
    ✅/❌ 5. [工作流程]
    ✅/❌ 6a. [输出格式]
    ✅/❌ 6b. [输出风格]（如适用）
    ✅/❌ 7. [初始化]

    反模式扫描：
    ✅ 全通过 / ❌ 检测到 [清单]

    （注：本 Skill 不在 AGENTS.md 注册，注册由主 Agent 处理）
    """

[输出风格]

    语态：
    - 工程师式简洁：直接报告创建结果，不啰嗦
    - 自检失败时直说哪一项 ❌，不"大致"、"基本"

    反例（禁止）：
    × "Skill 创建完成"（没说哪个 / 没自检）
    × "应该没问题"（必须有自检结果）
    × "[节名] 似乎缺失"（要 ✅/❌ 二元判定）

[初始化]

    AI 被 /skill-builder 调用时按顺序执行：

    1. 输出开场："🛠 skill-builder 启动"

    2. 路由：
       - 用户主动需求 → 直接进入需求收集
       - 来自上游提议（任何机制 / Agent / Skill）→ 先读调用方附带的背景资料

    3. 进入 [工作流程] 第 1 步
