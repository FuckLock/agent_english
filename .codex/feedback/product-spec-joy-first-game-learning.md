# product-spec-builder 快乐优先的游戏化英语学习

**问题描述**：在英语学习产品需求迭代中，AI 容易把文档写成普通学习工具、功能清单或 Provider 配置平台，弱化用户真正的产品初心：快乐学习、像打游戏一样推进、用图片降低文字枯燥感。

**触发场景**：用户补充：“在确定下我的初衷，我想要快乐的学习，所以类似于游戏一样，然后如果文字太枯燥呢所以加呢图片来打游戏，继续深入产品文档是否合理。”

**教训/建议**：product-spec-builder 处理此类英语学习产品时，应把“快乐学习 / 游戏化 / 图片驱动”放在产品定位和体验原则前部。Provider、模型、设置中心都是支撑能力，不应抢主体验。图片不是装饰，而是把英文文字转成副本封面、漫画分镜、怪兽卡、装备卡、通关卡等游戏场景的核心手段。验收指标也要包含快乐感、反枯燥和继续打下一关的意愿。
